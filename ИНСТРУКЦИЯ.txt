==== ИНСТРУКЦИЯ ПО ИСПОЛЬЗОВАНИЮ WIFI SPAMMER ====
(ТОЛЬКО ДЛЯ ОБРАЗОВАТЕЛЬНЫХ ЦЕЛЕЙ)

Данный набор инструментов предназначен исключительно для образовательных целей и демонстрации принципов работы беспроводных сетей. 

=== СОДЕРЖИМОЕ ПАКЕТА ===

1. wifi_spammer.py - основной скрипт для создания фейковых WiFi сетей
2. wifi_scanner.py - инструмент для сканирования существующих WiFi сетей
3. run_on_windows.bat - утилита для упрощенного запуска на Windows
4. README.md - подробная документация на английском языке
5. requirements.txt - список зависимостей
6. adapter_monitor_mode.md - инструкции по включению режима мониторинга

=== УСТАНОВКА ===

1. Установите Python 3.6 или выше
2. Установите Npcap (скачайте с https://npcap.com)
3. Откройте командную строку от имени администратора и выполните:
   pip install -r requirements.txt

=== ПОРЯДОК ИСПОЛЬЗОВАНИЯ ===

ШАГ 1: Подготовка
- Подключите WiFi адаптер, который поддерживает режим мониторинга
- Ознакомьтесь с файлом adapter_monitor_mode.md для информации о поддерживаемых адаптерах

ШАГ 2: Сканирование сетей (опционально)
- Запустите командную строку от имени администратора
- Выполните: python wifi_scanner.py -i "Имя_вашего_адаптера" -t 30
- Это покажет существующие WiFi сети в вашем окружении

ШАГ 3: Запуск WiFi Spammer
- Двойной клик на файле run_on_windows.bat, или
- В командной строке с правами администратора выполните:
  python wifi_spammer.py -i "Имя_вашего_адаптера" -n 15 -d 0.3

ШАГ 4: Остановка
- Нажмите Ctrl+C в командной строке для остановки скрипта

=== БЕЗОПАСНОСТЬ И ОТВЕТСТВЕННОСТЬ ===

ПОМНИТЕ: Использование этих инструментов может:
1. Нарушать локальное законодательство о радиосвязи
2. Создавать помехи для других пользователей WiFi
3. Привлечь внимание администраторов сети

Используйте только в контролируемой среде, например:
- В лаборатории 
- В домашних условиях на коротких расстояниях
- С согласия всех потенциально затрагиваемых лиц

=== УСТРАНЕНИЕ ПРОБЛЕМ ===

Если программа выдает ошибки:
1. Убедитесь, что вы запустили от имени администратора
2. Проверьте, правильно ли установлен Npcap
3. Убедитесь, что ваш WiFi адаптер поддерживает режим мониторинга
4. Попробуйте использовать другое имя интерфейса (можно узнать через команду netsh wlan show interfaces)
